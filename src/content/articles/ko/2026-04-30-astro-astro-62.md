## 실험적 커스텀 로거 기능

**Astro 6.2**에서는 개발자가 로깅 동작을 완전히 제어할 수 있는 실험적 커스텀 로거 기능을 도입했습니다. 이 기능은 특히 **JSON 출력**을 지원하여 로그 분석 도구나 모니터링 시스템과의 통합을 크게 개선합니다.

커스텀 로거를 설정하려면 `astro.config.mjs` 파일에서 다음과 같이 구성할 수 있습니다:

```javascript
import { defineConfig } from 'astro/config';

export default defineConfig({
  experimental: {
    customLogger: {
      // JSON 형식으로 로그 출력
      format: 'json',
      // 로그 레벨 설정
      level: 'info',
      // 커스텀 로그 핸들러
      handler: (log) => {
        console.log(JSON.stringify({
          timestamp: new Date().toISOString(),
          level: log.level,
          message: log.message,
          data: log.data
        }));
      }
    }
  }
});
```

이 기능은 대규모 프로젝트에서 로그 수집과 분석을 체계화하는 데 매우 유용하며, CI/CD 파이프라인에서 빌드 과정을 모니터링할 때 특히 효과적입니다.

## SVG 최적화 API

**Astro 6.2**는 새로운 **SVG 최적화 API**를 제공하여 SVG 파일의 크기를 줄이고 성능을 향상시킬 수 있습니다. 이 API는 빌드 시점과 런타임 모두에서 사용할 수 있으며, 다양한 최적화 옵션을 제공합니다.

기본적인 SVG 최적화는 다음과 같이 사용할 수 있습니다:

```javascript
import { optimizeSVG } from 'astro/svg';

const svgContent = `
  <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="40" fill="blue" />
  </svg>
`;

const optimized = await optimizeSVG(svgContent, {
  removeComments: true,
  removeUnusedNamespaces: true,
  minifyStyles: true,
  removeEmptyElements: true
});
```

컴포넌트에서 SVG 최적화를 활용하는 예시:

```astro
---
import { optimizeSVG } from 'astro/svg';

const iconSVG = await optimizeSVG(await Astro.glob('../icons/*.svg'), {
  removeViewBox: false,
  mergePaths: true
});
---

<div class="icon-container">
  {iconSVG.map(svg => (
    <Fragment set:html={svg.content} />
  ))}
</div>
```

## 폰트 파일 URL 헬퍼

새로운 **폰트 파일 URL 헬퍼** 기능은 웹폰트 관리를 대폭 개선합니다. 이 기능을 통해 폰트 파일의 경로를 자동으로 해결하고, 프리로딩 최적화를 쉽게 구현할 수 있습니다.

`getFontURL` 헬퍼를 사용한 예시:

```astro
---
import { getFontURL } from 'astro/fonts';

// 로컬 폰트 파일 URL 생성
const notoSansURL = getFontURL('/fonts/NotoSans-Regular.woff2');
const robotoURL = getFontURL('/fonts/Roboto-Bold.woff2');
---

<html>
  <head>
    <!-- 폰트 프리로딩 -->
    <link rel="preload" href={notoSansURL} as="font" type="font/woff2" crossorigin>
    <link rel="preload" href={robotoURL} as="font" type="font/woff2" crossorigin>
    
    <style>
      @font-face {
        font-family: 'Noto Sans';
        src: url({notoSansURL}) format('woff2');
        font-display: swap;
      }
      
      @font-face {
        font-family: 'Roboto';
        src: url({robotoURL}) format('woff2');
        font-weight: bold;
        font-display: swap;
      }
    </style>
  </head>
  <body>
    <h1 style="font-family: 'Roboto', sans-serif;">제목</h1>
    <p style="font-family: 'Noto Sans', sans-serif;">본문 내용</p>
  </body>
</html>
```

## 추가 개선사항 및 업그레이드 가이드

**Astro 6.2**에는 위의 주요 기능 외에도 여러 개선사항이 포함되어 있습니다:

- **타입스크립트 지원 강화**: 새로운 API들에 대한 완전한 타입 정의 제공
- **빌드 성능 최적화**: SVG 처리와 폰트 로딩에서 약 15% 성능 향상
- **개발 서버 안정성**: HMR 과정에서의 메모리 누수 문제 해결
- **에러 처리 개선**: 더 명확한 에러 메시지와 스택 트레이스 제공

업그레이드는 다음 명령어로 간단히 수행할 수 있습니다:

```bash
# npm 사용 시
npm update astro

# yarn 사용 시
yarn upgrade astro

# pnpm 사용 시
pnpm update astro
```

기존 프로젝트에서 새 기능을 활용하려면 `astro.config.mjs` 파일의 `experimental` 플래그를 활성화해야 합니다. 현재 버전에서는 Breaking Changes가 없으므로 기존 코드는 그대로 동작합니다.

개발팀은 이러한 실험적 기능들을 안전하게 테스트해보고, 피드백을 Astro 커뮤니티에 공유하여 향후 안정 버전으로의 전환에 기여할 수 있습니다.
