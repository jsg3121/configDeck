구글 지식·정보 부문 SVP인 Nick Fox가 AI 검색 기능만으로 매주 수십억 건의 클릭을 웹사이트에 보내고 있다고 LinkedIn과 X에서 밝혔다. Search Engine Journal은 이 수치에 기준선, 분모, 방법론이 전혀 동반되지 않았다는 점을 집중적으로 분석했다.

## 무엇이 새로운가

구글이 공식적으로 인용하는 수치는 두 가지다. 하나는 기존부터 언급해 온 "검색을 통해 매일 수십억 건의 클릭을 전송"이라는 일간 수치이고, 다른 하나는 이번에 처음 등장한 "AI 기능만으로 매주 수십억 건의 클릭"이라는 주간 수치다. 두 수치 모두 "수십억(billions)"이라는 단위만 제시되어 있어 AI 기능 클릭이 전체 검색 클릭에서 차지하는 비율을 알 수 없다. 구글이 Search Console에 도입 중인 생성형 AI 성능 보고서도 노출수만 제공하고 클릭 데이터는 포함하지 않는다. 즉, 개별 사이트 운영자가 Fox의 집계 수치를 자기 데이터와 대조할 방법이 현재로서는 없다.

## 설정 파일에 어떤 의미인가

이 뉴스는 개발 도구 설정이나 빌드 파이프라인에 직접 영향을 주는 내용이 아니다. 다만 웹사이트를 운영하면서 Search Console API를 활용해 트래픽을 모니터링하고 있다면, AI 검색 관련 노출 데이터가 별도 보고서로 분리되기 시작했다는 점은 알아 둘 만하다. 현재 AI 성능 보고서에는 클릭 수가 빠져 있으므로, 기존 Search Console 대시보드나 API 연동 스크립트에서 AI 트래픽의 실질적 유입 효과를 측정하기는 불가능하다. 구글은 향후 더 많은 지표를 AI 성능 보고서에 추가하겠다고 했지만, 어떤 지표가 언제 추가될지는 밝히지 않았다. 설정이나 연동 방식에 변화가 생기면 그때 다시 정리하겠다.

## 다음 단계 제안

Search Console에서 생성형 AI 성능 보고서 접근 권한이 있는지 확인해 보라. 아직 선별된 사이트 소유자에게만 제공 중이다. 접근이 가능하다면 노출수 추이라도 기록해 두는 것이 좋다. 클릭 데이터가 추가되는 시점에 비교 기준선을 확보할 수 있기 때문이다. 구글이 제시한 집계 수치 자체보다, 내 사이트에 실제로 어떤 변화가 있는지를 측정할 수 있는 인프라를 먼저 갖추는 편이 실용적이다.
