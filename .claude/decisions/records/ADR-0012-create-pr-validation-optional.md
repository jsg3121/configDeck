# ADR-0012: create-pr 스킬 검증 단계 선택적 실행

- 상태: 승인됨
- 날짜: 2026-04-14
- 의사결정자: 사용자, Claude

## 맥락 (Context)

기존 `create-pr` 스킬은 모든 PR 생성 시 린트, 코드 리뷰, 테스트 등의 검증 과정을 일괄 실행했다. 대상 브랜치에 따라 검증 수준이 달랐지만, 항상 최소한의 검증(린트 + 코드 리뷰)은 필수였다.

**문제점:**

1. **토큰 낭비**: 단순 버그 수정, 스타일 변경 등 간단한 PR에도 전체 검증 파이프라인이 실행되어 불필요한 토큰이 소비됨
2. **컨텍스트 낭비**: 코드 리뷰 서브에이전트 실행 시 하네스 재로딩 + 분석 결과 누적으로 컨텍스트 윈도우가 빠르게 소진됨
3. **시간 지연**: 간단한 PR 생성에도 검증 대기 시간이 발생하여 빠른 피드백 루프가 저해됨

**기존 플로우:**
```
상태 확인 → 자동 포맷팅 → ESLint 검사 → 코드 리뷰 → [테스트] → PR 생성
```

## 결정 (Decision)

`create-pr` 스킬의 검증 단계를 **사용자 선택 기반**으로 변경한다.

### 새로운 플로우

```
1. 상태 확인
2. 대상 브랜치 확인
3. 검증 옵션 선택 (AskUserQuestion, multiSelect)
   - □ 정적 분석 (ESLint, Prettier, TypeScript)
   - □ 코드 리뷰 (서브에이전트)
   - □ 단위 테스트 (Vitest)
   - □ E2E 테스트 (Playwright)
4. 선택된 검증만 실행
5. 이슈 발견 시 보고 + 계속 여부 확인
6. PR 생성
```

### 기본값 정책

| 대상 브랜치 | 기본 선택 |
|------------|----------|
| `main` | 정적 분석 + 단위 테스트 권장 (선택은 사용자 판단) |
| 그 외 | 모두 미선택 (검증 없이 바로 PR) |

### 추가 기능

- 사용자가 "Other"에 "전체 QA" 입력 시 `qa-agent` 호출 가능
- 검증 결과 요약 테이블 제공

## 근거 (Rationale)

### 1. 토큰 효율성

검증 옵션별 토큰 비용:

| 옵션 | 실행 방식 | 토큰 비용 |
|------|----------|----------|
| 정적 분석 | Bash 직접 실행 | 낮음 |
| 코드 리뷰 | 서브에이전트 | 중간~높음 |
| 단위 테스트 | Bash 직접 실행 | 낮음 |
| E2E 테스트 | Bash 직접 실행 | 중간 |

코드 리뷰 서브에이전트 1회 실행 시 하네스 로딩 + 코드 분석으로 수천 토큰이 발생한다. 단순 스타일 수정 PR에 이 비용을 지불할 필요가 없다.

### 2. 사용자 판단 존중

PR의 성격(신규 기능, 버그 수정, 리팩토링, 단순 수정)에 따라 필요한 검증 수준이 다르다. 사용자가 변경 내용을 가장 잘 알고 있으므로, 검증 범위를 사용자가 결정하는 것이 합리적이다.

### 3. 빠른 피드백 루프

단순 수정 시 검증 없이 바로 PR을 생성할 수 있어, 빠른 반복 작업이 가능해진다.

### 4. 유연성 유지

필요하면 전체 검증도 가능하고, `qa-agent`를 통한 종합 QA도 선택할 수 있어 품질 관리 옵션이 줄어들지 않는다.

## 대안 (Alternatives Considered)

| 대안 | 불채택 사유 |
|------|------------|
| 검증을 별도 스킬로 완전 분리 (`/create-pr` + `/qa`) | 두 스킬을 순차 호출해야 하는 번거로움. PR 생성 흐름 내에서 검증 옵션을 선택하는 것이 UX상 자연스러움 |
| 검증 레벨 파라미터 (`/create-pr --full`, `--lite`) | CLI 스타일이라 직관성 떨어짐. 체크박스 UI가 더 명확하게 선택 가능 |
| main 머지만 검증 필수, 나머지는 검증 없음 | 유연성 부족. 사용자가 feature 브랜치에서도 테스트를 돌리고 싶을 수 있음 |

## 결과 (Consequences)

### 긍정적 결과

- **토큰 절약**: 단순 PR에서 코드 리뷰 서브에이전트 호출 생략으로 토큰 50% 이상 절감 예상
- **속도 향상**: 검증 없이 PR 생성 시 10~60초 → 5초 이내로 단축
- **유연성**: 작업 성격에 맞는 검증 수준 선택 가능

### 주의사항

- main 머지 시 검증을 선택하지 않으면 품질 이슈가 발생할 수 있음 → 권장 메시지로 안내
- CI/CD에서 최종 품질 게이트가 있다면 이 변경의 리스크가 줄어듦

### 후속 조치

1. `.claude/skills/create-pr/SKILL.md` 업데이트 완료
2. CLAUDE.md 변경 이력에 기록 필요
