# ADR-0005: 공유 링크 기능 — URL 기반 옵션 인코딩

- 상태: 승인됨
- 날짜: 2026-04-06
- 의사결정자: jsg3121

## 맥락 (Context)

ConfigDeck은 서버 없는 정적 사이트(ADR-0002)로, 설정 파일 생성과 다운로드가 클라이언트에서 완결된다. 기획서에서 공유 링크 기능은 후순위(P2)로 분류되어 있었다.

그러나 반응형 전략을 논의하는 과정에서, 모바일에서의 활용 시나리오가 도출되었다. 개발자가 모바일에서 옵션을 선택하고 공유 링크를 생성하여 메신저 등에 저장해두면, 이후 PC에서 해당 링크로 접속하여 다운로드하는 플로우가 가능하다. 이를 통해 모바일 대응의 실용적 가치가 확보된다.

## 결정 (Decision)

공유 링크 기능을 **URL 쿼리스트링 기반**으로 구현한다. 서버 없이 클라이언트에서 완결되는 방식을 유지한다.

### 사용자 플로우

```
[모바일] 옵션 선택 → 공유 링크 생성 → 메신저/메모에 저장
[PC] 링크 접속 → 옵션 자동 복원 → 미리보기 확인 → 다운로드/복사
```

### 화면별 역할

- **모바일**: 옵션 선택 + 공유 링크 생성 (미리보기는 간소화 또는 생략)
- **데스크톱**: 옵션 선택 + 실시간 미리보기 + 다운로드/복사/ZIP + 공유 링크 생성

### 구현 방식

선택된 옵션을 URL 쿼리 파라미터로 인코딩한다.

```
https://config-deck.dev/generator?stack=react,vite,ts&eslint=strict&prettier=semi:false
```

옵션이 많아져 URL이 길어지는 경우 base64 인코딩 등 압축 방식을 검토한다.

## 근거 (Rationale)

- 서버 없이 클라이언트에서 완결되므로 기존 아키텍처(ADR-0002, ADR-0004)와 정합
- URL만으로 상태가 복원되므로 로그인/DB가 불필요
- 모바일에서의 실용적 활용 시나리오를 확보하여 반응형 대응의 투자 가치를 높임
- 기획서의 "공유 링크 생성(후순위)" 기능과 자연스럽게 연결됨

## 대안 (Alternatives Considered)

| 대안 | 불채택 사유 |
|------|------------|
| 서버 사이드 단축 URL | DB/API 서버 필요, 정적 사이트 아키텍처에 부합하지 않음 |
| localStorage 저장 후 QR 코드 | 디바이스 간 전달이 번거로움, QR 생성 라이브러리 추가 필요 |
| 모바일 미대응 (데스크톱 전용) | 모바일 검색 유입을 활용하지 못함, SEO 전략과 상충 |

## 결과 (Consequences)

- 생성기 페이지에서 "공유 링크 복사" 기능을 제공한다
- URL 파라미터 스키마를 정의하고, 페이지 진입 시 파라미터를 파싱하여 옵션을 복원하는 로직이 필요하다
- 모바일 UI에서는 미리보기 영역을 간소화하고 옵션 선택 + 공유에 집중한다
- 반응형 전략은 모바일 퍼스트를 유지한다
