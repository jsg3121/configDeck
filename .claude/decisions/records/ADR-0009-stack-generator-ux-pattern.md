# ADR-0009: 스택 생성기 UX 패턴 선정 (아코디언 인라인 옵션)

- 상태: 승인됨
- 날짜: 2026-04-13
- 의사결정자: jsg3121, Claude

## 맥락 (Context)

ConfigDeck의 스택 생성기는 프레임워크 단위(React+Vite+TS, Next.js 등)로 여러 설정 파일을 한 번에 생성하는 기능이다. 기존에는 파일별로 프리셋만 지정하고 개별 옵션 커스터마이즈가 불가능했다.

SPEC-0001(옵션 생성기 기능 보강)에서 파일별 옵션 시스템(OptionControl 판별 유니온, 7종 입력 컨트롤, core/advanced tier)을 도입한 후, 스택 생성기에서도 이 옵션 시스템을 활용하여 파일별 세부 옵션을 조정할 수 있어야 한다는 요구가 생겼다.

문제는 스택 생성기가 **여러 파일을 동시에** 다루므로, 파일당 10~30개의 옵션을 한 화면에 어떻게 보여줄지 UX 설계가 필요했다.

## 결정 (Decision)

**B안(아코디언 인라인 옵션 + 우측 미리보기)**을 채택한다.

### 레이아웃 구조

```
┌─────────────────────────────────────────────────┐
│  스택 프리셋 탭 (수평 버튼)                        │
├────────────────────┬────────────────────────────┤
│  좌측: 아코디언     │  우측: 파일 탭 + 코드 미리보기 │
│  ☑ .gitignore      │  [.gitignore] [tsconfig] ...│
│  ☑ tsconfig.json ▾ │                            │
│    프리셋: [드롭다운]│  {                          │
│    target: [ES2022] │    "compilerOptions": {     │
│    strict: ☑       │      ...                    │
│  ☑ eslint.config ▸ │  }                          │
│  ☑ prettier ▸      │                            │
├────────────────────┴────────────────────────────┤
│  [ZIP 다운로드]  [파일별 복사]                     │
└─────────────────────────────────────────────────┘
```

### 핵심 동작

1. 아코디언은 **한 번에 하나만** 열린다
2. 아코디언 안에는 **core 옵션만** 표시한다 (파일별 생성기 페이지로의 링크로 전체 옵션 유도)
3. 아코디언을 열면 우측 미리보기 탭도 해당 파일로 자동 전환
4. 스택 프리셋 선택 시 각 파일에 지정된 프리셋 값이 초기값으로 적용
5. 사용자가 옵션을 변경하면 해당 파일의 미리보기가 실시간 갱신

## 근거 (Rationale)

### 외부 레퍼런스 조사 (RES-0002)

7개 서비스(Spring Initializr, Create T3 App, createapp.dev, Astro, Nx Console, Yeoman, PureDevTools)를 조사한 결과:

- **createapp.dev**: ConfigDeck과 가장 유사한 구조 (다중 파일 생성 + 파일 트리 + 실시간 미리보기). 단, 파일별 세부 옵션 조정은 불가
- **PureDevTools**: 프리셋 버튼 탭 + 섹션별 세부 옵션 + 실시간 미리보기. 단일 파일 전용
- **Create T3 App**: 조건부 옵션 (이전 선택에 따라 관련 옵션만 표시)

B안은 createapp.dev의 파일 탐색 구조와 PureDevTools의 인라인 옵션 조정을 결합한 패턴이다.

### B안이 적합한 이유

1. **주요 시나리오에 최적화**: 스택 사용자의 90%는 "프리셋으로 빠르게 시작 + 1~2개 파일만 옵션 미세 조정". 아코디언은 이 플로우에 최적
2. **인지 부하 최소화**: 한 번에 하나의 파일 옵션만 펼쳐지므로 전체 옵션이 압도적으로 보이지 않음
3. **실시간 피드백**: 아코디언에서 옵션 변경 → 우측 미리보기 즉시 반영. 변경 효과를 바로 확인 가능
4. **기존 구현 재활용**: 파일별 생성기의 OptionForm 컨트롤(7종)을 아코디언 내부에 그대로 재활용 가능

## 대안 (Alternatives Considered)

| 대안 | 불채택 사유 |
|------|------------|
| **A안: 좌측 파일 리스트 + 우측 미리보기/옵션 탭 전환** | 옵션 수정 시 "옵션" 탭으로 전환해야 하는 추가 클릭. 미리보기와 옵션을 동시에 볼 수 없음 |
| **C안: 탭 기반 파일별 전환** | 3패널 구조가 복잡. 모바일 대응 어려움. 파일 간 전환 시 컨텍스트 유실 |
| **프리셋 고정 (커스터마이즈 불가)** | ConfigDeck의 핵심 가치(옵션 커스터마이즈)와 상충. createapp.dev, Create T3 App과 차별점 없음 |

## 결과 (Consequences)

- `StackGenerator.svelte` 리팩터링 필요: 아코디언 내부에 파일별 OptionForm(core 옵션만) 렌더링
- `stacks.ts`는 프리셋 참조 방식 유지 (이미 리팩터링 완료): 초기값은 프리셋에서, 사용자 변경은 touchedKeys로 추적
- 좌측 아코디언 너비를 충분히 확보해야 함 (w-96 이상). select/number 컨트롤이 좁은 영역에서 사용하기 어려울 수 있음
- 모바일에서는 아코디언과 미리보기가 상하 배치로 전환되어야 함
