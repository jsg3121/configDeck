# ADR-0007: 페이지 구조 재설계 — 파일/스택별 생성기 통합 및 FAQ 홈 통합

- 상태: 승인됨
- 날짜: 2026-04-06
- 의사결정자: jsg3121

## 맥락 (Context)

ADR-0006에서 프리셋 페이지를 생성기 내로 통합했으나, 여전히 구조적 문제가 남아 있었다:

1. **생성기 메인이 만능 페이지**: 모든 파일의 옵션 설정과 미리보기를 한 페이지에서 처리하면, 페이지가 과도하게 비대해지고 SEO 대응이 어려움
2. **파일별/스택별 랜딩이 중간 페이지에 불과**: 설명만 보여주고 생성기로 이동시키는 구조는 불필요한 전환 단계
3. **FAQ가 별도 페이지**: 내용이 적은데 독립 페이지로 존재할 이유가 부족

## 결정 (Decision)

### 1. 파일별 랜딩 + 생성기 통합

기존에 분리되어 있던 "파일별 SEO 랜딩"과 "생성기의 파일 옵션 설정"을 하나의 페이지로 통합한다.

```
/generator/eslint-config    → ESLint 설명 + 옵션 설정 + 실시간 미리보기
/generator/tsconfig         → TSConfig 설명 + 옵션 설정 + 실시간 미리보기
/generator/prettier-config  → Prettier 설명 + 옵션 설정 + 실시간 미리보기
```

각 페이지가 독립적인 SEO 랜딩이면서 동시에 생성기 기능을 제공한다.

### 2. 스택별 랜딩 + 생성기 통합

스택별 페이지도 동일하게 통합한다. 여러 파일을 묶어서 생성하는 기능을 포함한다.

```
/generator/react-vite-ts    → React+Vite+TS 설명 + 관련 파일 옵션 + 미리보기
/generator/nextjs           → Next.js 설명 + 관련 파일 옵션 + 미리보기
```

### 3. 생성기 메인을 허브 페이지로 변경

`/generator`는 직접 생성 기능이 아닌, 파일/스택 카드 목록과 프리셋 선택을 제공하는 허브 역할로 변경한다.

### 4. FAQ를 홈 하단 섹션으로 통합

FAQ를 별도 페이지에서 제거하고, 홈 페이지 하단에 서비스 소개 흐름의 일부로 배치한다.

### 5. 최종 페이지 구조 (6개 유형)

| 페이지 | 라우트 | 역할 |
|--------|--------|------|
| 홈 | `/` | 서비스 소개, 이용 방법, FAQ, 생성기 CTA |
| 생성기 허브 | `/generator` | 파일/스택 카드 목록, 프리셋 선택 → 하위 페이지 이동 |
| 파일별 생성기 | `/generator/{file-name}` | 2분할 (옵션 + 미리보기), SEO 랜딩 겸용 |
| 스택별 생성기 | `/generator/{stack-name}` | 여러 파일 묶어서 생성, SEO 랜딩 겸용 |
| 문서/가이드 | `/docs` | 학습형 콘텐츠 |
| 블로그 | `/blog` | 검색 유입용 |

## 근거 (Rationale)

1. **파일별 페이지 = 생성기**: "eslint config generator" 검색 → `/generator/eslint-config` 진입 → 바로 옵션 설정 가능. 중간 안내 페이지 없이 즉시 가치 제공
2. **SEO와 기능의 자연스러운 결합**: 각 파일/스택 페이지가 검색 키워드에 대응하면서 동시에 생성 기능을 제공하므로, 유입된 사용자의 전환율이 높아짐
3. **생성기 허브의 역할 명확화**: 전체 파일/스택을 탐색하고 선택하는 "카탈로그" 역할에 집중
4. **FAQ 홈 통합**: 별도 페이지를 만들 만큼의 콘텐츠 양이 아니며, 홈에서 서비스 소개와 함께 노출하는 것이 전환에 유리

## 대안 (Alternatives Considered)

| 대안 | 불채택 사유 |
|------|------------|
| 기존 구조 유지 (랜딩 → 생성기 이동) | 불필요한 페이지 전환. 검색 유입 사용자가 바로 기능을 사용할 수 없음 |
| 생성기 메인에서 모든 파일 처리 | 페이지 비대화, 파일별 SEO 대응 불가, URL 공유 시 특정 파일 지정 불가 |
| FAQ 별도 페이지 유지 | 콘텐츠 양 대비 페이지 비용이 높음. 홈에 통합하는 것이 사용자 동선에 자연스러움 |

## 결과 (Consequences)

- 기획서의 IA, 페이지 구조, 내비게이션, 사용자 플로우 전면 수정 필요
- `/files/{name}`, `/stacks/{category}/{name}` 라우트 제거 → `/generator/{name}`으로 통합
- FAQ 페이지 라우트 제거, 글로벌 내비게이션에서 FAQ 메뉴 제거
- 파일별/스택별 생성기 페이지는 공통 레이아웃(2분할)을 공유하되, 옵션 스키마만 다르게 구성
