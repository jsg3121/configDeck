# ADR-0006: 생성기 중심 IA 재설계

- 상태: 승인됨
- 날짜: 2026-04-06
- 의사결정자: jsg3121

## 맥락 (Context)

초기 기획서(configDeckIA.md)는 홈, 생성기, 프리셋, 파일별 랜딩, 스택별 랜딩 등 다수의 페이지를 별도로 구성하는 IA를 설계했다. UI/UX 레퍼런스 조사(RES-0001)를 통해 유사 서비스들의 패턴을 분석한 결과, 사용자가 가장 빠르게 결과에 도달할 수 있는 구조로 재설계가 필요하다고 판단했다.

특히 PureDevTools TSConfig Generator의 "프리셋 → 세부 옵션 → 실시간 미리보기" 2단계 플로우가 ConfigDeck의 핵심 가치(빠름, 조합 가능)에 가장 부합한다.

## 결정 (Decision)

### 1. 프리셋 페이지 통합

프리셋을 별도 페이지(`/presets`)로 분리하지 않고, 생성기 페이지 내 프리셋 버튼으로 통합한다. 프리셋 선택 시 해당 옵션들이 자동으로 채워지고 미리보기가 동시에 반영된다.

### 2. 사용자 플로우 단순화

기존 4단계 플로우(파트 → 스택 → 파일 → 옵션)를 2단계로 단순화한다:
- **경로 A**: 프리셋 선택 → 옵션 조정 (필요 시)
- **경로 B**: 직접 선택 → 옵션 조정

### 3. UI 레이아웃

PureDevTools 스타일의 2분할 패널을 채택한다:
- **좌측**: 설정 파일 선택 + 옵션 폼 (체크박스, 드롭다운, 토글 등)
- **우측**: 실시간 코드 미리보기 + 복사/다운로드 액션
- **모바일**: 상하 배치로 전환

### 4. IA 구조 변경

변경 전:
```
/ (홈) → /generator (생성기) → 옵션 설정 → 결과
/ (홈) → /presets (프리셋) → 선택 → 커스터마이징 → 결과
```

변경 후:
```
/ (홈: 서비스 소개 + CTA)
/generator (생성기: 프리셋 + 직접 선택 + 옵션 + 미리보기 통합)
/files/{name} (파일별 SEO 랜딩 → 생성기 진입)
/stacks/{category}/{name} (스택별 SEO 랜딩 → 생성기 진입)
```

## 근거 (Rationale)

1. **PureDevTools TSConfig Generator**가 프리셋 → 세부 옵션 → 실시간 미리보기를 단일 페이지에서 제공하며, 이 패턴이 가장 빠른 결과 도달을 보여줌
2. **프리셋 = 옵션 자동 선택**으로 설계하면, 프리셋과 직접 선택이 동일한 인터페이스에서 자연스럽게 공존
3. 별도 프리셋 페이지는 사용자가 "프리셋 선택 → 이동 → 확인 → 커스터마이징"의 추가 전환 단계를 거쳐야 하여, 핵심 가치인 "빠름"과 충돌
4. 2분할 레이아웃은 설정 변경의 결과를 즉시 확인할 수 있어 사용자 확신을 높임

## 대안 (Alternatives Considered)

| 대안 | 불채택 사유 |
|------|------------|
| 기획서 원안 (프리셋 별도 페이지) | 페이지 전환으로 인한 플로우 단절. 프리셋 선택 후 다시 생성기로 이동해야 하는 비효율 |
| Spring Initializr 스타일 (메타데이터 폼 + 의존성 검색) | ConfigDeck은 프로젝트 메타데이터가 없고, 설정 파일 옵션이 핵심이므로 검색 기반보다 직접 선택이 적합 |
| 스텝 위자드 (ScaffoldHub 스타일) | 단계별 진행이 자유로운 탐색을 제한하며, 옵션 변경 시 이전 단계로 돌아가야 하는 번거로움 |

## 결과 (Consequences)

- 기획서(configDeckIA.md)의 IA, 페이지 구조, 사용자 플로우 섹션 수정 필요
- `/presets` 라우트 제거, 생성기 페이지에 프리셋 기능 통합
- 네비게이션에서 "Presets" 메뉴 제거 또는 생성기 내 앵커 링크로 변경
- 파일별/스택별 SEO 랜딩 페이지는 유지하되, CTA가 생성기로 직접 연결
