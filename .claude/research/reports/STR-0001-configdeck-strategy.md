## 상황 요약 (Context)

### 핵심 시장 기회

1. **ESLint v9 flat config 전환 혼란** — 2024년 4월 공식화 이후 마이그레이션 수요 폭증. "eslint flat config generator" + "eslint flat config migration" 키워드가 사실상 무주공산. 생성과 마이그레이션 두 축으로 SEO 앵커 확장 가능
2. **스택 번들 config 생성기라는 시장 공백** — 기존 서비스는 모두 단일 파일 특화(gitignore.io, PureDevTools 등), 여러 파일을 한 번에 생성하는 서비스 부재
3. **비영어권 다국어 SEO 블루오션** — 한국어/일본어 config 생성기 전무
4. **Vite 생태계 급성장** — 주간 8,900만 다운로드, 전용 config 생성기 없음
5. **AI 도구 설정 파일이라는 신규 수요** — Cursor(.cursorrules), Copilot(copilot-instructions.md), Claude Code(CLAUDE.md) 등 AI 도구 자체의 설정 파일 생성 수요 급증. AI를 대체재에서 보완재로 전환 가능

### 핵심 위협

1. **AI 코드 생성 도구의 급속 발전** (Copilot 매출 248% 성장) — 12~18개월 내 config 생성 품질 개선 가능. 단, AI 도구 설정 파일 지원으로 보완재 관계 형성 시 위협 완화 가능
2. **ESLint v9 기회창의 시한성** — 전환 완료 후 수요 감소 불가피
3. **낮은 진입 장벽** — 정적 사이트로 구현 가능하여 모방 용이

### 자사 강점

- 극한의 저비용 구조 (Cloudflare Pages 무료, 서버 비용 0원)
- 최신 기술 스택 (Astro + Svelte 5 + TypeScript + Tailwind v4)
- Claude Code 활용 빠른 개발 가속
- 한국어 네이티브 + 영어 가능

### 제약 조건

- 1인 개발 대역폭 한계
- 브랜드 인지도 및 Domain Authority 제로
- 애드센스 수익은 초기 트래픽 단계에서 미미 (MAU 1만 기준 월 $60~150)
- 일본어 네이티브 역량 없음

## 옵션별 상세 분석

### 옵션 A: ESLint 앵커 집중 전략

- **방향**: ESLint v9 flat config 생성기 + 마이그레이션 도구를 킬러 피처로 가장 먼저 출시. "eslint flat config generator"와 "eslint flat config migration" 두 키워드로 SEO 1위를 목표로 집중 투자. 이후 트래픽을 기반으로 다른 파일(prettier, tsconfig, gitignore)로 확장
- **전제 조건**: ESLint v9 마이그레이션 혼란이 최소 6개월 이상 지속. 관련 키워드의 경쟁 강도가 낮은 상태 유지
- **리스크**: (1) ESLint 공식 팀이 자체 웹 생성기/마이그레이션 도구를 출시할 경우 직접 경쟁 발생 (2) 전환 완료 후 키워드 검색량 자연 감소 (3) 단일 파일 앵커 의존으로 gitignore.io와 같은 한계에 봉착
- **기대 효과**: 3~6개월 내 특정 키워드 SEO 상위권 진입. "생성"과 "마이그레이션" 두 축으로 SEO 키워드 폭이 넓어짐. ESLint 사용률 89.3%라는 거대한 타겟 풀에 직접 접근. 마이그레이션은 도구 업데이트 시 재방문을 유도하여 리텐션도 개선

### 옵션 B: 스택 번들 차별화 전략

- **방향**: "프로젝트 시작 = ConfigDeck"이라는 인식을 목표로, 스택별 생성기(React+Vite+TS 등)를 핵심 가치로 전면 배치. 파일 간 충돌 자동 해결을 USP로 강조
- **전제 조건**: 스택 번들 생성에 대한 사용자 니즈가 실제 전환(다운로드)으로 이어짐. 파일 간 충돌 해결 로직 구현이 MVP 일정 내 완료 가능
- **리스크**: (1) MVP 범위가 넓어져(4개 파일 x 스택별 조합) 론칭 지연 (2) "스택 번들"이라는 컨셉이 SEO 키워드로 명확하지 않아 검색 유입이 약할 수 있음 (3) 충돌 해결 로직의 복잡도가 예상보다 높을 수 있음
- **기대 효과**: 시장 내 유일한 포지셔닝 확보. 경쟁자가 모방하기 어려운 복합 가치(번들+충돌해결+설명) 구축. CLI 스캐폴더(create-t3-app 등)의 웹 대안으로 자리매김

### 옵션 C: 다국어 니치 선점 전략

- **방향**: 한국어를 1차 언어로 서비스 완성 후, 한국 개발자 커뮤니티(velog, disquiet, 카카오 오픈채팅 등)에서 먼저 인지도 확보. 이후 영어, 일본어로 확장
- **전제 조건**: 한국어 config 생성기 검색 수요가 실재. 한국 개발자 커뮤니티에서의 바이럴이 초기 트래픽을 만들어낼 수 있음
- **리스크**: (1) 한국어 시장 규모가 작아 MAU 천장이 낮음 (2) 글로벌 확장 시 영어 SEO를 처음부터 다시 쌓아야 함 (3) 한국어 우선이 "로컬 서비스"라는 인식을 굳힐 수 있음
- **기대 효과**: 경쟁 제로인 시장에서 즉시 1위 확보. 한국어 네이티브 역량으로 콘텐츠 품질 보장. 커뮤니티 피드백으로 빠른 제품 개선

### 옵션 D (하이브리드): ESLint 앵커 + 한국어 선공략

- **방향**: 옵션 A와 C를 결합. ESLint flat config 생성기 + 마이그레이션을 킬러 피처로 하되, 영어+한국어 동시 론칭으로 두 시장을 동시 공략. MVP 범위는 핵심 4개 파일(.gitignore, eslint, prettier, tsconfig)로 제한하되 스택 번들 1~2개도 포함
- **전제 조건**: 영어+한국어 동시 론칭이 개발 일정을 크게 지연시키지 않음 (Astro i18n 구조가 이미 설계됨). ESLint v9 기회창이 유효
- **리스크**: (1) 두 시장 동시 공략으로 어느 쪽도 깊이 있게 파고들지 못할 가능성 (2) 다국어 콘텐츠 유지 비용이 1인 체제에서 부담될 수 있음
- **기대 효과**: 영어 SEO(ESLint 앵커)와 한국어 SEO(경쟁 부재) 양면에서 트래픽 확보. 한 쪽이 부진해도 다른 쪽이 보완. 다국어 구조를 MVP부터 검증하여 일본어 확장 기반 마련

## 권고 방향

### 권고 옵션 및 이유

**권고 옵션: D (하이브리드) — ESLint 앵커 + 한국어 선공략**

1. **시간 제약에 최적화된 전략이다.** ESLint v9 기회창(6~12개월)을 활용하면서, 한국어라는 경쟁 부재 시장에서 즉시 성과를 확보할 수 있다. 두 축이 서로의 리스크를 헤지한다.

2. **1인 개발 대역폭에 적합하다.** MVP를 핵심 4개 파일로 축소하고, Astro의 i18n 라우팅 구조가 이미 설계되어 있으므로 다국어 추가의 한계 비용이 낮다. 스택 번들은 MVP에서 1~2개(React+Vite+TS, Next.js)로 제한하여 번들 가치를 검증한다.

3. **SEO 전략이 명확하다.** 영어권에서는 "eslint flat config generator", "eslint flat config migration", "eslint config generator" 등 특정 고의도 키워드를 공략하고, 한국어에서는 "eslint 설정", "eslint 마이그레이션", "prettier 설정 파일", "gitignore 생성기" 등 경쟁 부재 키워드를 광범위하게 커버한다.

4. **검증 후 확장이 자연스럽다.** MVP 검증 후 옵션 B(스택 번들 강화)로 확장하거나, 한국어 성공 시 일본어(옵션 C 확장)로 자연스럽게 이어진다.

단일 옵션 A나 C만 선택하지 않는 이유: A만 선택하면 ESLint 기회창이 닫힐 때 대안이 없고, C만 선택하면 글로벌 시장 기회를 놓친다. ConfigDeck의 다국어 구조가 이미 기획 단계에서 설계되어 있으므로 D의 추가 비용이 크지 않다.

| 1인 개발로 90일 내 MVP 론칭이 가능하다 | Claude Code 활용에도 생성 규칙 데이터 구축, 충돌 해결 로직 등이 예상보다 복잡할 수 있음 | MVP 범위를 엄격히 제한 (4개 파일, 스택 번들 1~2개). 충돌 해결은 가장 빈번한 케이스만 우선 처리 |
| SEO DA 0에서 3~6개월 내 유의미한 순위 확보가 가능하다 | 경쟁 키워드에서 기존 고DA 사이트에 밀릴 수 있음 | 롱테일 키워드(파일별+프레임워크별 조합)와 한국어 키워드에서 먼저 순위 확보. 영어 핵심 키워드는 6개월 이상 투자 |

## 전략적 피봇 트리거

다음 조건 발생 시 전략 재검토가 필요하다:

1. **AI 피봇 트리거**: Copilot/Cursor가 ESLint v9 flat config를 정확하게 생성하는 것이 확인되면, (1) "생성기"에서 "검증기/최적화기"로 포지션 전환 검토 및 (2) AI 도구 설정 파일(.cursorrules 등) 지원을 앞당겨 AI 보완재 전략 가속
2. **ESLint 기회창 소멸 트리거**: ESLint v9 관련 키워드 검색량이 전월 대비 30% 이상 감소하면, 앵커를 AI 도구 설정 파일, Vite config 또는 스택 번들로 전환
3. **번들 가치 부재 트리거**: 론칭 후 60일간 ZIP 다운로드가 전체 다운로드의 5% 미만이면, 스택 번들 투자를 중단하고 파일별 생성기에 집중
4. **한국어 시장 부진 트리거**: 론칭 후 60일간 한국어 유입이 전체의 5% 미만이면, 다국어 투자를 축소하고 영어 SEO에 집중
