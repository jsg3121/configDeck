# ConfigDeck 서비스 기획서

> **최종 수정**: 2026-04-06
> **관련 ADR**: ADR-0006 (생성기 중심 IA 재설계), ADR-0007 (페이지 구조 재설계)

---

## 1. 서비스 개요

### 서비스명

ConfigDeck

### 한 줄 소개

개발자가 프로젝트 시작 시 필요한 설정 파일을 스택 기반으로 조합·생성·다운로드할 수 있는 다국어 웹 서비스

### 서비스 목적

개발자가 신규 프로젝트를 시작할 때 반복적으로 작성하는 설정 파일을 빠르게 생성할 수 있도록 하여:

- 초기 세팅 시간을 줄이고
- 설정 누락을 방지하며
- 최신 권장 방식 기반의 설정을 쉽게 적용할 수 있도록 돕는다

### 핵심 가치

| 가치 | 설명 |
|------|------|
| **빠름** | 설정 파일을 검색하고 복붙하는 시간을 줄임 |
| **정확함** | 최신 스택 기준으로 파일 구조를 제공 |
| **조합 가능** | 체크박스와 프리셋으로 필요한 옵션만 반영 |
| **다국어 지원** | 해외 개발자도 쉽게 사용 가능 |
| **확장성** | 단일 파일 생성 → 프로젝트 세팅 번들 생성으로 확장 가능 |

## 2. 문제 정의

### 기존 문제

신규 프로젝트 생성 시 개발자가 자주 겪는 문제:

1. 어떤 설정 파일이 필요한지 매번 다시 찾아봐야 함
2. 같은 설정을 여러 프로젝트에서 반복 작성함
3. ESLint, Prettier, TypeScript 등 서로 연관된 설정의 조합이 번거로움
4. `.gitignore`처럼 단순해 보이는 파일도 OS, IDE, 프레임워크에 따라 달라짐
5. 공식 문서의 "최소 설정"과 "실전용 설정" 사이 간극이 있음

### 해결 방향

사용자가 파일/스택을 선택하고 옵션을 체크하면, 해당 조합에 맞는 설정 파일들을 생성하여 개별 복사 또는 ZIP 다운로드할 수 있도록 한다.

## 3. 타겟 사용자

### 1차 타겟

**주 타겟**

- 프론트엔드 개발자
- 신규 프로젝트를 자주 세팅하는 개발자
- 개인/사이드 프로젝트를 자주 시작하는 개발자
- 템플릿보다 세부 옵션 조정이 필요한 개발자

**세부 타겟**

| 사용자 | 니즈 |
|--------|------|
| 주니어 개발자 | 무엇을 넣어야 할지 잘 모름 → 프리셋과 설명으로 가이드 |
| 미드/시니어 개발자 | 빠르게 표준 세팅을 만들고 싶음 → 직접 선택 + 미리보기 |
| 프리랜서/에이전시 개발자 | 반복 세팅이 많음 → 프리셋 + 공유 링크 |
| 팀 리드 | 팀 표준 세팅을 만들고 싶음 → 프리셋 + 공유 링크 |

### 2차 타겟

- 백엔드 개발자
- Android / iOS 개발자
- 개발 교육/부트캠프 수강생

## 4. 서비스 범위

### MVP 범위

초기 버전은 가장 수요가 높고 텍스트 기반 설정 파일 생성에 적합한 영역부터 시작한다.

**지원 파일**

| 카테고리 | 파일 |
|---------|------|
| 공통 | `.gitignore`, `.editorconfig`, `.env.example` |
| 프론트엔드 / Node | `tsconfig.json`, `eslint.config.mjs`, `prettier.config.mjs`, `vite.config.ts`, `vitest.config.ts`, `next.config.js` |

**지원 기능**

- 파일별 옵션 선택 + 실시간 미리보기
- 프리셋 3~5개
- 코드 복사 / 파일 다운로드 / ZIP 다운로드
- 영어 / 한국어

### 확장 범위

| 카테고리 | 대상 |
|---------|------|
| 백엔드 | NestJS, Spring Boot, Python/FastAPI, ASP.NET, Go |
| 모바일 | Android, iOS |
| DevOps | Docker, Docker Compose, GitHub Actions, CI/CD 템플릿 |

## 5. 서비스 컨셉

### 핵심 컨셉

> "설정 파일 생성기"가 아니라 **"프로젝트 초기 세팅 빌더"**

파일 하나만 만드는 것이 아니라, 스택 전체를 기준으로 관련 파일을 묶어서 제공하는 것이 차별점이다.

### 예시 시나리오

사용자가 **React + Vite + TS** 프리셋을 선택하면, 아래 파일을 한 번에 생성:

- `.gitignore`
- `.editorconfig`
- `tsconfig.json`
- `eslint.config.mjs`
- `prettier.config.mjs`
- `vite.config.ts`
- `vitest.config.ts`

## 6. 핵심 기능 정의

### 6-1. 파일 생성 기능

사용자가 옵션을 선택하면 해당 조합에 맞는 설정 파일을 생성한다.

- 개별 파일 코드 보기
- 복사하기
- 파일 단건 다운로드
- 전체 ZIP 다운로드

### 6-2. 옵션 조합 기능

파일별로 세부 옵션을 체크박스, 드롭다운, 토글 등의 UI로 제공한다.

**옵션 예시**

| 파일 | 주요 옵션 |
|------|----------|
| `.gitignore` | OS (macOS/Windows), IDE (VS Code/JetBrains), 프레임워크, 사용자 지정 폴더 |
| `eslint.config.mjs` | JS/TS, 프레임워크(React/Vue/Next/Node), import 정렬, 엄격도, Prettier 연동 |
| `prettier.config.mjs` | single quote, trailing comma, semi, print width, tab width |
| `tsconfig.json` | strict 여부, path alias, module resolution, target/lib |

### 6-3. 프리셋 기능

많이 쓰는 조합을 미리 정의하여 프리셋 버튼으로 제공한다. 프리셋 선택 시 해당 옵션들이 자동으로 채워지고 미리보기에 즉시 반영된다.

**MVP 프리셋**

- React + Vite + TypeScript
- Next.js + TypeScript
- Astro + TypeScript
- Node API + TypeScript

### 6-4. 설명 기능

각 옵션과 파일이 왜 필요한지 설명을 제공한다.

- 이 옵션을 켜면 무엇이 달라지는지
- 어떤 환경에서 주로 쓰는지
- 다른 옵션과 충돌하는지
- 권장 조합은 무엇인지

### 6-5. 다국어 기능

- UI, 설명, 문서 번역
- 다국어 SEO 페이지 운영
- **생성 파일 내부 키/패키지명/명령어는 원문 유지**

## 8. 페이지 구조 상세

### 8-1. 홈 (`/`)

**목적**: 서비스 소개, 이용 방법 안내, 생성기 진입점

**구성**

| 섹션 | 내용 |
|------|------|
| 히어로 | 서비스 한 줄 소개 + "설정 파일 만들기" CTA |
| 인기 프리셋 | 프리셋 카드 목록. 클릭 시 생성기 허브로 이동하여 해당 프리셋 적용 |
| 이용 방법 | 3단계 설명 (파일 선택 → 옵션 설정 → 다운로드) |
| 지원 파일 목록 | 파일별 생성기 페이지로 바로가기 |
| FAQ | 자주 묻는 질문 (별도 페이지 없이 홈 하단에 배치) |

**주요 CTA**

- "설정 파일 만들기" → `/generator`
- "프리셋으로 시작하기" → `/generator` (프리셋 사전 선택)

### 8-2. 생성기 허브 (`/generator`)

**목적**: 파일/스택별 생성기 목록을 탐색하고 선택하는 카탈로그

**구성**

| 섹션 | 내용 |
|------|------|
| 프리셋 | 프리셋 버튼 목록. 클릭 시 해당 스택별 생성기 페이지로 이동 |
| 파일별 카드 | 지원하는 설정 파일 카드 목록. 클릭 시 `/generator/{file-name}`으로 이동 |
| 스택별 카드 | 스택 프리셋 카드 목록. 클릭 시 `/generator/{stack-name}`으로 이동 |

### 8-3. 파일별 생성기 (`/generator/{file-name}`)

**목적**: 개별 설정 파일의 옵션 설정 + 코드 생성. SEO 검색 유입 랜딩 겸용

**레이아웃**: PureDevTools 스타일 2분할

```
┌─────────────────────────┬─────────────────────────┐
│  좌측 패널 (설정)        │  우측 패널 (미리보기)    │
│                         │                         │
│  파일 설명 (간략)        │  생성된 코드             │
│  프리셋 버튼 (관련 것만)  │  (실시간 반영)           │
│  옵션 폼               │                         │
│   ├ 체크박스            │  파일명 표시             │
│   ├ 드롭다운            │  코드 하이라이팅          │
│   ├ 토글               │                         │
│   └ 텍스트 입력         │  ┌─────────────────┐    │
│                         │  │ 복사 │ 다운로드   │    │
│  고급 옵션 (접힘)        │  └─────────────────┘    │
└─────────────────────────┴─────────────────────────┘
```

**반응형**: 모바일에서는 상하 배치 (설정 → 미리보기)

**SEO 요소**: 페이지 상단에 파일 설명, 사용 환경, FAQ 등 검색 유입용 콘텐츠 포함

### 8-4. 스택별 생성기 (`/generator/{stack-name}`)

**목적**: 스택에 필요한 여러 파일을 묶어서 생성. SEO 검색 유입 랜딩 겸용

**레이아웃**: 파일별 생성기와 동일한 2분할이되, 좌측에 여러 파일의 옵션이 아코디언으로 구성

```
┌─────────────────────────┬─────────────────────────┐
│  좌측 패널 (설정)        │  우측 패널 (미리보기)    │
│                         │                         │
│  스택 설명 (간략)        │  파일 탭                 │
│  포함 파일 체크리스트     │  ┌──────┬──────┬─────┐ │
│                         │  │.git  │tscon │esli │ │
│  ▼ .gitignore 옵션      │  └──────┴──────┴─────┘ │
│  ▼ tsconfig.json 옵션   │                         │
│  ▼ eslint.config 옵션   │  선택된 파일의 코드       │
│  ▼ prettier.config 옵션 │  (실시간 반영)           │
│                         │                         │
│                         │  ┌─────────────────┐    │
│                         │  │ 복사 │ ZIP 다운  │    │
│                         │  └─────────────────┘    │
└─────────────────────────┴─────────────────────────┘
```

**반응형**: 모바일에서는 상하 배치

### 8-5. 문서/가이드 (`/docs`)

**목적**: 학습형 유입 확보 및 신뢰도 강화

**예시 문서**

- ESLint flat config란?
- `.gitignore`에 무엇을 넣어야 하나?
- React 프로젝트 초기 세팅 체크리스트
- TypeScript strict 모드 추천값
- Prettier와 ESLint 함께 쓰는 방법

### 8-6. 블로그 (`/blog`)

**목적**: 검색 유입 확보

**예시 주제**

- `eslint.config.mjs` 예제 모음
- Next.js에서 추천하는 tsconfig 구성
- Android 프로젝트용 `.gitignore` 예시

## 9. 사용자 플로우

### 플로우 A: 직접 생성 (파일 단위)

```
홈 → "설정 파일 만들기" → 생성기 허브 → 파일 카드 클릭
→ 파일별 생성기 (/generator/eslint-config)
→ 옵션 설정 (좌측) ↔ 실시간 미리보기 (우측)
→ 복사 또는 다운로드
```

### 플로우 B: 프리셋 사용 (스택 단위)

```
홈 → "프리셋으로 시작하기" → 생성기 허브 → 스택 카드 클릭
→ 스택별 생성기 (/generator/react-vite-ts)
→ 포함 파일 확인 + 옵션 조정 (좌측) ↔ 파일 탭 미리보기 (우측)
→ ZIP 다운로드
```

### 플로우 C: 검색 유입

```
"eslint config generator" 검색
→ /generator/eslint-config 직접 진입 (SEO 랜딩)
→ 상단 설명 확인
→ 옵션 설정 → 미리보기 → 다운로드
```

### 옵션 충돌

| 상황 | 처리 |
|------|------|
| ESLint + Prettier 규칙 충돌 | Prettier 연동 옵션 선택 시 충돌 규칙 자동 비활성화 + 안내 메시지 |
| tsconfig의 `module`과 `moduleResolution` 불일치 | 의존 관계에 따라 자동 보정 + 경고 표시 |
| 프리셋 선택 후 개별 옵션 변경 | 프리셋 상태 해제되고 "커스텀" 상태로 전환. 원래 프리셋으로 복원 버튼 제공 |

### 빈 상태

| 상황 | 처리 |
|------|------|
| 아무 파일도 선택하지 않음 | 미리보기 영역에 "파일을 선택하세요" 안내 표시 |
| 아무 옵션도 선택하지 않음 | 기본값(권장값)으로 생성. 미리보기에 기본 설정 표시 |
| 스택별 생성기에서 모든 파일 체크 해제 | ZIP 다운로드 버튼 비활성화 + 안내 메시지 |

### 모바일 제약

| 상황 | 처리 |
|------|------|
| 화면이 좁아 2분할 불가 | 상하 배치로 전환. 미리보기는 하단에 접힌 상태로 제공, 탭으로 전환 |
| 코드 미리보기가 길어짐 | 스크롤 영역으로 제한. "전체 보기" 버튼으로 모달/전체화면 전환 |
| ZIP 다운로드가 모바일에서 불편 | 공유 링크 복사를 우선 노출하여 데스크톱에서 이어서 작업할 수 있도록 유도 |

### URL/공유 링크

| 상황 | 처리 |
|------|------|
| 공유 링크의 옵션이 더 이상 존재하지 않음 | 유효하지 않은 옵션은 무시하고 나머지만 적용 + "일부 옵션이 변경되었습니다" 안내 |
| URL이 과도하게 길어짐 | 옵션을 Base64 인코딩하여 압축 (ADR-0005 참조) |

## 15. 차별화 포인트

### 기존 생성기와의 차이

단순히 파일 하나를 만드는 것이 아니라:

- 스택 기준으로 묶어서 생성
- 파일 간 충돌을 자동으로 해결
- 최신 권장 방식 반영
- 각 옵션에 대한 설명 제공
- 다국어 지원
- 파일/스택별 SEO 랜딩 겸용 구조

### 핵심 메시지

> "개발 설정 파일 하나를 만드는 서비스"가 아니라
> **"프로젝트 시작에 필요한 설정을 한 번에 정리해주는 서비스"**

- 언어별 유입 비중
