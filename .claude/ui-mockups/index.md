# ui-mockups/

UI 시안(HTML 모킹) 보관 폴더이다. ui-publisher 에이전트가 코드 작성 전에 시안을 먼저 만들고, 사용자 검토를 거친 후 실제 코드로 적용한다.

## 이 폴더의 역할

- 코드 작성 전 디자인 의도를 시각적으로 검증할 수 있는 HTML 시안 보관
- 디자인 의사결정 기록 (어떤 레이아웃/색상/구조를 왜 채택했는지)
- 시안 ↔ 실제 코드의 연결 추적 (작업 폴더의 README.md에 기록)
- 향후 동일 영역 재작업 시 디자인 결정 근거를 다시 확인할 수 있는 자료

## 폴더 구조

```
.claude/ui-mockups/
├── index.md                          # 이 파일
├── {작업명}/                         # 작업 단위 폴더 (kebab-case)
│   ├── README.md                     # 시안 의도, 디자인 결정, 변경 이력, 적용 코드 경로
│   ├── {시안}.html                   # 시안 파일 (단일 HTML + Tailwind CDN)
│   └── {시안-다른상태}.html          # 상태별 별도 시안 (필요 시)
└── ...
```

### 작업 단위 폴더 명명 규칙

- kebab-case로 작성 (예: `ai-config-catalog`, `header-dropdown`)
- 가능하면 SPEC/ADR 번호와 영역명을 조합 (예: `spec-0005-ai-config-catalog`)

## 작성 규칙

- 모든 시안 작성 워크플로우는 [conventions/guides/ui-publishing.md](../conventions/guides/ui-publishing.md)를 따른다
- 시안은 단일 HTML 파일 + Tailwind CDN으로 작성하여 사용자가 IDE/브라우저로 즉시 확인할 수 있게 한다
- 인터랙션은 상태별 별도 HTML + 어노테이션 주석으로 표현한다
- 작업 폴더에 README.md를 두어 시안 의도와 디자인 결정을 기록한다

## 시안 폴더 README.md 필수 항목

- **시안 의도**: 어떤 페이지/컴포넌트의 시안인지, SPEC/ADR 연결
- **디자인 결정**: 주요 레이아웃/색상/타이포그래피 결정과 근거
- **시안 파일 목록**: 각 파일이 어떤 상태/뷰포트를 표현하는지
- **변경 이력**: 사용자 피드백에 따른 시안 갱신 기록
- **적용 코드 경로** (적용 후 추가): 시안이 실제 어느 파일로 구현되었는지 + 시안 대비 차이
