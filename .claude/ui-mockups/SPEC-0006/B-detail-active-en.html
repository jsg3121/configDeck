Summary

Next.js 13.0.0 through 15.2.2 contain a middleware authorization bypass: an attacker can supply a forged x-middleware-subrequest header to skip authentication/authorization middleware and reach protected routes (e.g. /admin) directly. Upgrade to a patched version (14.2.25, 15.2.3, or later) immediately.

Vulnerability mechanism

The Next.js middleware runtime uses the x-middleware-subrequest header to identify internal sub-requests (e.g. follow-up of rewrites or redirects). This header was intended to be internal-only, but external requests could also supply it, and Next.js had a code path that skipped middleware execution when it was present.

As a result, an attacker could attach the header to a request for a protected route and reach the underlying handler without going through authentication or authorization middleware. The CVSS 9.1 (Critical) rating reflects the immediacy and breadth of the impact.

Patch and remediation

1. Upgrade to a patched version

# Upgrade to the patched release on your major
npm install next@14.2.25
# or upgrade to the latest stable
npm install next@latest

2. Temporary mitigation (before patching)

If you cannot patch immediately, explicitly reject the x-middleware-subrequest header at the entry of your middleware: 

// middleware.ts
import { NextResponse } from 'next/server'

export function middleware(request) {
  if (request.headers.get('x-middleware-subrequest')) {
    return new NextResponse(null, { status: 400 })
  }
  // ... your existing auth logic
}
