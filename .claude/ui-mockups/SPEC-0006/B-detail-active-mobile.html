요약

Next.js 13.0.0 ~ 15.2.2의 미들웨어에서 x-middleware-subrequest 헤더를 외부에서 조작하여 인증/인가 검사를 우회할 수 있습니다. 즉시 패치 권장.

호스팅 환경별 영향

  • · Vercel Edge Middleware: 영향받음
  • · Netlify: 영향받음
  • · Cloudflare Pages: 별도 확인 필요

취약점 메커니즘

Next.js 미들웨어 런타임은 내부 서브요청 식별을 위해 x-middleware-subrequest 헤더를 사용합니다. 외부에서 이 헤더를 위조하면 미들웨어 본문 실행이 건너뛰어 인증 우회가 발생합니다.
