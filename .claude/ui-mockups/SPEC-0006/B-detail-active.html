Next.js 미들웨어 인증 우회 (CVE-2025-29927)

요약

Next.js 13.0.0 ~ 15.2.2의 미들웨어에서 x-middleware-subrequest 헤더를 외부에서 조작하여 전달하면 인증/인가 검사를 우회할 수 있습니다. 미들웨어로 보호된 관리자 페이지나 보호된 API 라우트가 인증 없이 접근될 수 있어 즉시 패치 버전(14.2.25, 15.2.3 등)으로의 업그레이드가 권장됩니다. Vercel/Netlify 등 호스팅 환경 사용자는 즉각 영향 평가가 필요합니다.

취약점 메커니즘

Next.js의 미들웨어 런타임은 내부 서브요청(예: rewrites, redirects 후속 처리)을 식별하기 위해 x-middleware-subrequest 헤더를 사용합니다. 본래 이 헤더는 내부 통신용으로 외부에서 전달될 수 없어야 하지만, 실제로는 외부 요청에서도 전달이 가능했고 Next.js는 이 헤더가 있을 때 미들웨어 본문 실행을 건너뛰는 분기를 가지고 있었습니다.

결과적으로 공격자는 보호된 라우트(예: /admin)에 해당 헤더를 임의로 추가하여 인증/인가 미들웨어 자체를 통과하지 않고 핸들러에 도달할 수 있게 됩니다. CVSS 9.1(Critical)이 부여된 이유는 인증 우회의 즉시성과 광범위한 영향 때문입니다.

패치 및 해결 방안

1. 패치 버전으로 업그레이드

# 사용 중인 메이저에 맞는 패치 버전으로 업그레이드
npm install next@14.2.25
# 또는 최신 안정 버전
npm install next@latest

2. 임시 완화 방안 (패치 적용 전)

패치를 즉시 적용할 수 없는 경우, 미들웨어 진입 지점에서 x-middleware-subrequest 헤더를 명시적으로 거부하는 방어 로직을 추가합니다. 

// middleware.ts
import { NextResponse } from 'next/server'

export function middleware(request) {
  if (request.headers.get('x-middleware-subrequest')) {
    return new NextResponse(null, { status: 400 })
  }
  // ... 기존 인증 로직
}
