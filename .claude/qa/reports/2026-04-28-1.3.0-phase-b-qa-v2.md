## 차단 이슈 (머지 전 수정 필요)

### [E2E] generator-flow.spec.ts mobile-chrome — networkidle 대기 누락

- **위치**: `tests/e2e/flows/generator-flow.spec.ts:7`, `tests/e2e/flows/generator-flow.spec.ts:38`
- **브라우저**: Mobile Chrome 단독 실패 (Chromium, Firefox, WebKit, Mobile Safari 통과)
- **실패 건수**: 2건
- **에러 메시지**:
  ```
  테스트 1 (line 7):
  Error: expect(locator).toBeVisible() failed
  Locator: getByRole('region', { name: '생성된 코드' }).locator('pre code')
  Expected: visible
  Timeout: 5000ms

  테스트 2 (line 38):
  Test timeout of 30000ms exceeded.
  Error: locator.textContent: Test timeout of 30000ms exceeded.
  waiting for getByRole('region', { name: '생성된 코드' }).locator('pre code')
  ```
- **원인 분석**: Q3-B 커밋(5a56323)에서 `pre code` 셀렉터를 `getByRole('region', { name: '생성된 코드' }).locator('pre code')` ARIA 기반으로 교체하고 networkidle 대기를 추가했으나, `generator-flow.spec.ts`의 두 테스트에는 `networkidle` 대기가 누락되었다. 같은 파일의 `검색 기능 플로우`(line 74)와 `모바일 뷰 플로우`(line 132)에는 `await page.waitForLoadState('networkidle')`이 추가되어 통과하는 반면, `홈 → 생성기 → ESLint 설정 생성 플로우`(line 7)와 `프리셋 선택 및 옵션 변경 플로우`(line 38)에는 hydration 대기가 없어 Svelte 아일랜드가 마운트되기 전에 `pre code` 요소를 탐색하여 실패한다.
- **수정 제안**: 두 테스트에 `previewTab.click()` 이후 또는 `page.goto()` 이후 `await page.waitForLoadState('networkidle')` 추가.
  ```typescript
  // 테스트 1 (line 7): goto 후 또는 previewTab.click() 이후 추가
  await page.goto('/ko/generator/eslint-config')
  await page.waitForLoadState('networkidle')  // 추가 필요

  // 테스트 2 (line 38): goto 이후 추가
  await page.goto('/ko/generator/eslint-config')
  await page.waitForLoadState('networkidle')  // 추가 필요
  ```
