## 차단 이슈 (머지 전 수정 필요)

### [E2E] pre code strict mode violation — generator 테스트 회귀

- **위치**: `tests/e2e/pages/generator.spec.ts:60`, `tests/e2e/flows/generator-flow.spec.ts:29` 외 다수
- **브라우저**: Chromium, Firefox, Mobile Chrome (Mobile Safari, WebKit 통과)
- **실패 건수**: 25건 (고유 테스트 12개 × 브라우저 조합)
- **에러 메시지**:
  ```
  Error: strict mode violation: locator('pre code') resolved to 2 elements:
      1) <code>export default [↵]</code> aka getByText('export default [ ]')
      2) <code>{↵  "fileSlug": "eslint-config",↵  "locale": "ko"…</code>
  ```
- **원인 분석**: Phase B 5~6단계에서 ESLint 생성기 페이지에 `MigrationPanel.svelte`가 추가되었다. 이 컴포넌트가 내부적으로 별도의 `pre code` 블록(마이그레이션 결과 또는 입력 미리보기)을 렌더링하여 페이지 내 `pre code` 요소가 2개가 되었다. Playwright의 기본 `page.locator()`는 strict mode이므로 2개 이상의 요소가 매칭되면 실패한다.
  - **회귀 확인**: 1.2.0 QA 재검증(2026-04-26 22:55) 시점 E2E 0건 실패였으나, feature/1.3.0 머지 이후 현 시점에서 25건으로 증가.
  - Mobile Safari, WebKit은 동일 케이스 통과 — Mobile Safari는 일부 케이스가 skip 처리됨.
- **수정 제안**: `page.locator('pre code')` → `page.locator('.code-preview pre code').first()` 또는 generator 전용 컨테이너 선택자로 범위 한정. 또는 Playwright `locator.first()`로 첫 번째 요소만 단언.

## 권장 이슈

### [E2E] home.spec.ts:24 — header strict mode 환경 의존성

- **위치**: `tests/e2e/pages/home.spec.ts:24`
- **설명**: `page.locator('header')` 가 7개의 요소를 감지하여 strict mode violation 발생. Astro DevToolbar 패널(개발 환경에서 주입)이 내부에 `header` 요소를 포함하는 것으로 추정된다. `pnpm preview`(프로덕션 빌드 서빙)를 사용하므로 DevToolbar는 포함되지 않아야 하나, Astro 버전에 따라 preview 환경에서도 toolbar가 렌더링될 수 있다.
- **수정 제안**: `page.locator('header')` → `page.locator('header.site-header')` 또는 더 구체적인 셀렉터 사용. 또는 `playwright.config.ts`에서 `devtools: false`를 명시.
