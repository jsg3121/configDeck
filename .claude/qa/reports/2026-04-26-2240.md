**파일**: `src/lib/utils/shareUrl.ts` (38, 51번 줄)

```
TS6387: 'unescape' is deprecated.  (line 38)
TS6387: 'escape' is deprecated.    (line 51)
```

- **원인**: `btoa(unescape(encodeURIComponent(str)))` / `decodeURIComponent(escape(atob(base64)))` 패턴이 레거시 API를 사용한다. `unescape`/`escape`는 HTML 스펙에서 Annex B legacy feature로 분류되며, TypeScript가 deprecated로 표시한다.
- **수정 방안**:
  - A) `TextEncoder`/`TextDecoder` + `Uint8Array` 기반 방식으로 교체 (Web API 표준, 브라우저 호환성 우수):
    ```ts
    const base64UrlEncode = (str: string): string =>
      btoa(String.fromCharCode(...new TextEncoder().encode(str)))
        .replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '')
    ```
  - B) `encodeURIComponent`/`decodeURIComponent` 자체가 UTF-8 인코딩을 처리하므로, `btoa`와 결합 시 직접 변환 가능한 방식 채택.
  - 현재 코드는 ASCII/UTF-8 범위에서 정상 동작하지만, `escape`/`unescape`의 deprecated 특성과 멀티바이트 문자 처리 엣지 케이스를 고려해 교체를 권장한다.

## 마이그레이션 로직 심층 분석 (요청된 주의 영역)

### src/lib/migration/ — 정규식 기반 변환 엣지 케이스

**eslintAuditor.ts — `extractRules` 정규식 취약성**

```ts
const rulesMatch = code.match(/["']?rules["']?\s*:\s*\{([^}]+)\}/s)
```

`[^}]+` 패턴은 중첩된 `}` 를 허용하지 않는다. 예:
```js
rules: {
  "no-restricted-syntax": ["error", { selector: "ForInStatement" }]
}
```
위 케이스에서 `{ selector: "ForInStatement" }` 의 `}` 에서 매칭이 끊겨 `rulesMatch`가 잘못된 범위를 캡처하거나 null을 반환한다. 결과적으로 audit이 규칙 목록을 누락할 수 있다.

**eslintAuditor.ts — `extractRules` 정규화 취약성**

```ts
.replace(/(\w+):/g, '"$1":')
```

이 패턴은 URL(`http:`) 이나 문자열 값 내부의 `:` 도 변환할 수 있다.

**parser.ts — `parseCommonJsConfig` 키 따옴표 처리**

```ts
.replace(/(['"])?(\w+)(['"])?\s*:/g, '"$2":')
```

`snake_case` 키(예: `parser-options`) 나 하이픈이 포함된 키는 `\w+`에 매칭되지 않아 파싱 실패한다.

**권장**: 정규식 기반 파싱을 `@babel/parser` 또는 `acorn` 등 실제 JS 파서로 교체하거나, 최소한 위 엣지 케이스에 대한 단위 테스트를 추가하여 회귀를 방지한다.
